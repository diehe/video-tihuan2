# 绿幕视频替换桌面软件设计

## 目标

构建一个 Mac 桌面 App，让用户选择两个本地视频：

- 主体视频：手机屏幕区域为绿色绿幕。
- 替换视频：需要合成到主体视频的手机绿幕区域中。

用户完成选择后，软件自动识别绿幕区域，允许用户用一个手动限定区域避免误扣其他绿色物体，并导出合成后的 MP4。

## 非目标

- 不再使用动作追踪作为主流程。
- 不再依赖 AI 关键帧校准。
- 不做云端上传或账号系统。
- 第一版不做批量任务队列。

## 产品形态

第一版是 Mac 桌面 App，沿用 Tauri + React + Python sidecar 架构：

- Tauri 负责桌面壳、本地文件选择、启动 Python 引擎。
- React 负责三步向导界面和预览交互。
- Python + OpenCV 负责逐帧绿幕检测、替换视频透视贴合、融合和导出。
- FFmpeg 负责音频 mux 和最终 MP4 编码。

## 用户流程

1. 选择主体视频和替换视频。
2. 软件读取主体视频首帧，自动检测绿色区域并显示 mask 预览。
3. 用户可拖动一个矩形限定区域，只在该区域内检测绿色，避免画面中其他绿色物体被替换。
4. 用户可选择基础合成参数：适配方式、音频来源、边缘羽化、mask 收缩/扩张。
5. 用户点击导出，软件逐帧扣绿并生成 MP4。

## 合成算法

每一帧执行：

1. 在限定区域内将 BGR 转 HSV。
2. 按绿色阈值生成 mask。
3. 使用形态学开闭运算清理噪点。
4. 根据配置对 mask 做收缩/扩张和羽化。
5. 找最大绿色轮廓，估算四角区域。
6. 将替换视频当前帧按四角区域透视变形到主体帧坐标系。
7. 只用处理后的绿色 mask 做 alpha 混合，保留手机边框、手指、非绿色前景和背景。
8. 如果某帧未检测到有效绿幕，复用上一帧四角但 mask 仍以当前帧绿色像素为准；如果连续失败，保留原帧并记录警告。

## 界面设计

三步向导：

- 输入：主体视频、替换视频、导出路径。
- 确认绿幕：首帧预览、绿色 mask 叠加、限定区域拖拽、参数轻量调节。
- 导出：进度状态、完成路径、失败提示。

默认隐藏高级参数，但保留这些设置：

- 画面适配：填满裁剪、拉伸填满、完整留边。
- 音频：保留主体视频、使用替换视频、静音。
- 边缘羽化：默认 3px。
- Mask 调整：默认轻微收缩 1px。

## API 设计

Python 引擎提供：

- `POST /chroma/analyze`：读取首帧，返回帧预览、默认限定区域、绿色 mask 覆盖率和候选四角。
- `POST /chroma/preview`：按给定时间和参数返回单帧合成预览。
- `POST /chroma/render`：按参数导出完整 MP4。

保留旧接口作为兼容，但前端新流程只调用 chroma 接口。

## 测试策略

Python：

- 用合成视频测试自动检测绿幕。
- 用包含绿色干扰物的视频测试限定区域只扣手机区域。
- 用单帧测试 mask 羽化和收缩不会污染非绿色区域。
- 用导出测试验证生成 MP4、帧数和音频策略。

前端：

- 测试 reducer 状态流：选择视频后可分析，分析后可预览和导出。
- 测试 API payload 使用 chroma 接口，不再要求 tracking。
- 测试 App 基本渲染和主要按钮文案。

## 成功标准

- 用户只需选择两个视频，默认参数即可导出。
- 有绿色干扰物时，用户能通过限定区域避免误替换。
- 主流程不需要动作追踪、AI key 或关键帧编辑。
- `pytest`、`npm test`、`npm run build` 通过。
